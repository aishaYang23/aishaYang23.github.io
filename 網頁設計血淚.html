~網頁骨架之建立~  
      1. 建構iframe

        2. 要將子網頁都放入主網頁，首先，我先設立一個iframe目標與位址，將子網頁連結顯示區都導引到所創立的框架中。

      2. 目錄製作

        3. 架設完畢子網頁所顯示的目標區域後，接下來的大工程是目錄navigation的製作，使點擊區塊後可產生網頁顯現的效果。此外，我也使用了hover & focus的程式語言，使讀者在使用目錄時，會因滑鼠游標的位置以及所點網頁的不同，而使目錄文字型態有所不同，使讀者可以知道目前所處網頁為何。

      3. 網頁配色

        4. 在設計網頁的過程中，配色也是非常重要且耗時費神的步驟之一，要將畫面配置的和諧且舒適實在不容易。在這次的網頁中，我所使用的主要是以紅色與粉紅色為主題，背景、文字、特效與超連結顏色主要如下：#d1b8c5，#b7756c，#E5BFBC，#CB8A90，#BC8D73，#FCE9DA，#9873B9。

~製作困難的克服~  
      1. 中文字體的繕打

        2. 起初，剛將網路上的簡單模板下載下來，準備製作網頁的設計時，還沒開始，就遇到了第一個大難題。那便是無論如何都無法將中文字體顯示在頁面上。一開始，我以為是字型設定的原因，因此將所有字體都改成中文字體，像是標楷體、華康少女體......等。然而，這一切的努力都毫無效果，因此便只好上網查詢解決方針。或許是因為第一次從頭到尾獨自利用程式語言設計網頁的關係，抑或是網路資訊太過繁雜，投靠古歌大神仍不見效。幸運地，有一群一同修課的好朋友٩(๑>◡<๑)۶，如及時雨解決了我的煩惱，原來是要在程式語言中打上：<meta charset="utf-8"/>這段程式碼，便可使中文字乖乖地顯示出來了。

      2. iframe大小調整

        3. 本網站之設立為多個子網頁，依照游標點擊，分別顯示於網站上所建立的iframe中。然而，每個子網頁的大小與長度並不相同，甚至差異頗大，因此有了希望iframe可以依照不同子網站的大小，自動調整相對應大小的念頭。不過，經過多方嘗試，網路上的各種方法似乎都無法達到預期的效果，此外，思及網頁中的留白，似乎也別是一種設計，因此最終就放下了這樣的執著，未再繼續找尋這個問題的解方了。

      3. github的嘗試

        4. 建立完畢所有網站內容後，接下來就是要將網站發布公開，如此一來，大家都能在沒有程式語言檔案的情況下瀏覽網頁，解決了檔案太多太大的問題。因此，藉由網路上的兩段創建網頁的影片：看完就懂GitHub & git、GitHub基本教學--從無到有，以及同學的幫助與討論，我嘗試了自行創建網站，利用網路上已架設好的網域，將自己的資料與檔案放入其中，如此一來便能獲取網頁連結，供所有人閱覽。此外，我也從中學到了幾個有趣的程式語言：git add. & git commit-m "COMMENT" & git push & git status的使用方式。雖然最中得知並非以這種方法進行網站架設，不過仍然是個有趣的體驗與嘗試。

      4. 網頁預設值的改寫

        5. 起先，我要更改範例網頁中預設值的footer顏色時，一直找不到方向，更改任何顏色也全無效果(⋟﹏⋞)。後來才發現，原來預設的背景是圖片，怪不得無法做出改變，找出了根本原因後，最中便很順利地解決了版面配色的問題。

      5. 並列排版的困境

        6. 起先，在進行照片排版時，不知如何將數張圖片進行並排，使用center or float都仍然無法達到想要的效果。就在不知究竟該如何是好時，我學到了一項方便好用的工具，亦即display: flex，他可以使圖片自由地排列，依照預期的格式進行變動。

      6. 表格製作

        7. 雖然老師上課時，已將表格的製作講解地非常詳細，然而，要親自從零打出一份完整的表格仍使我耗費了不少心力，尤其在跨欄標題與rowspan & colspan的使用上，有些複雜，然而成功研究完畢後，不但受益良多，同時也獲得滿滿的成就感。

      7. 圖層問題

        8. 在製作彈跳視窗與圖片動畫時，我遇到了另一個問題，那便是彈跳視窗跳出後的框框外黑屏，竟然會部分無法遮住照片動畫。這時候我才發現，原來網頁中圖層也是有前後順序的呢！誰在前面，誰在後面決定了是否會互相覆蓋，或是特效是否會互相影響。因此，藉由這次困境，我又學會了一項新的好工具，那便是z-index的程式語言用法，可利用此語言進行順序的排列，解決遮蔽的問題~！
